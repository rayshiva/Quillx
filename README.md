## 📘 What Quillx Does

Quillx is an open standard for showing when AI helped with software work. It uses the language of authorship, so teams can describe who wrote what in a plain way.

It is built for people who want clear project records without making value claims. Quillx helps you state AI involvement with care and consistency.

## 🧭 First Use

When Quillx opens, you can begin by reviewing the authorship fields in the app. These fields help you describe how AI took part in the work.

A simple first setup may include:

- Project name
- Human author name
- AI tool name
- Type of help given
- Date of disclosure

Keep the text short and clear. Quillx is made for plain disclosure, not for debate.

## 🧩 How to Use Quillx

Quillx follows a simple workflow:

1. Open your project record.
2. Add the human and AI authorship details.
3. Choose the role of each contributor.
4. Review the disclosure text.
5. Save or export the record for your project files.

You can use Quillx for:

- Software projects
- Documentation
- Release notes
- Team records
- Internal project files

The goal is to make authorship easier to read and easier to share.

## 🔍 Why This Standard Helps

Many teams want a clean way to show AI involvement without turning it into a moral label. Quillx gives you that structure.

It can help you:

- Keep project records clear
- Show where AI assisted the work
- Use the same format across projects
- Reduce guesswork for readers
- Keep disclosure simple for non-technical users

Quillx treats disclosure as a record of authorship. It does not try to judge the work.

## 🛠️ Common Tasks

### Add AI Involvement to a Project

Use the authorship fields to name the AI tool and explain what it helped with. Keep the wording direct. For example, you can note drafting, editing, or code help.

### Review a Disclosure

Before you save, check that each field is easy to understand. A reader should be able to tell:

- Who did the work
- What the AI helped with
- How much help it gave

### Share the Record

You can include the Quillx record in your project folder, release notes, or other project files. This makes the disclosure easy to find later.

## 📄 Example Use

- The team records both roles in Quillx
- The final project files include the disclosure

This keeps the record short and useful.

## 🧠 Best Practices

Use short, plain text. Write what happened, not what you think it means.

Good habits include:

- Name the AI tool used
- State the kind of help it gave
- Keep the wording consistent
- Update the record when the project changes
- Let the disclosure match the project facts

This makes the record useful for readers, teams, and future reviews.
